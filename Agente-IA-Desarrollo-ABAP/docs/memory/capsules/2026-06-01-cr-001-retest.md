## Invariant aparecido

**`pandoc` debe estar en el `PATH` del usuario ANTES de arrancar Claude Code, no después**. La verificación que hace el slash command `/validar-fd` para `.docx` es `command -v pandoc` en su propio subproceso bash. Si Claude Code hereda un `PATH` viejo (sin pandoc), la verificación devuelve `PANDOC_NO_DISPONIBLE` y la validación del `.docx` aborta antes de invocar al sub-agente.

Corolarios para el setup de la máquina del desarrollador:

1. Instalar pandoc **antes** que cualquier sesión Claude Code esté abierta.
2. Después de instalar, **abrir terminal nueva** para que `$env:Path` se refresque.
3. Para uso productivo, registrar la ruta del package winget en el `PATH` user-level con `[Environment]::SetEnvironmentVariable("Path", ...)`. winget **no** crea siempre el symlink en `WinGet\Links\` para hacerlo accesible — depende de la versión y de los privilegios del usuario.
4. Si winget falla, el `.msi` oficial de pandoc (`https://github.com/jgm/pandoc/releases/latest`) sí registra `pandoc.exe` en el PATH del sistema y se hereda a todos los procesos.

## Validación útil

### Generación reproducible del dataset multi-formato sin LaTeX instalado

Para generar `.pdf` desde `.md` sin instalar LaTeX/wkhtmltopdf/weasyprint/tectonic, usar Microsoft Edge headless (disponible por default en Windows 10+):

```bash
pandoc fd-base.md --standalone -o fd-base.html
"C:/Program Files (x86)/Microsoft/Edge/Application/msedge.exe" \
  --headless --disable-gpu --no-margins \
  --print-to-pdf="$(pwd -W)/fd-base.pdf" \
  "file:///$(pwd -W)/fd-base.html"
```

Pandoc + Edge cubren los 4 formatos del dataset sin dependencias adicionales:

| Formato | Generación |
|---|---|
| `.md` | Copia canónica desde `qa/tests/fixtures/fds/fd-completo-alv.md` |
| `.txt` | `pandoc fd-base.md -t plain -o fd-base.txt` |
| `.docx` | `pandoc fd-base.md -o fd-base.docx` |
| `.pdf` | HTML intermedio + Edge `--print-to-pdf` |

### Diagnóstico de `pandoc not found` en sesiones Claude Code

Si el slash command falla con `PANDOC_NO_DISPONIBLE` pero `pandoc --version` anda en la PowerShell del usuario:

1. La sesión Claude Code arrancó antes de que `pandoc` estuviera en el `PATH`.
2. Cerrar Claude Code completamente.
3. Verificar que `pandoc --version` funciona en una terminal **nueva**.
4. Desde **esa misma terminal** (donde pandoc anda), arrancar `claude`. La sesión hereda el `PATH` correcto.

### Verificación de la conversión `.docx → .md`

Aunque el slash command hace la conversión internamente al validar `.docx`, vale la pena tener una conversión fuera-de-banda para confirmar que pandoc preserva las 7 secciones obligatorias del FD genérico:

```bash
pandoc fd-base.docx -f docx -t markdown -o fd-base-from-docx.md
grep -cE "^## " fd-base-from-docx.md   # esperado: 7
```
